# Game Design Document Template

## GDD Structure

```markdown
# [Game Title] — Game Design Document

## 1. Overview
- **Genre:** [e.g., 2D Roguelike RPG]
- **Platform:** [e.g., Mobile (iOS/Android), PC]
- **Target Audience:** [e.g., Casual gamers 18-35]
- **Unique Selling Point:** [One sentence — what makes this game special]
- **Elevator Pitch:** [2-3 sentences describing the game]

## 2. Core Gameplay Loop
[Diagram: What does the player do every 30 seconds / 5 minutes / 1 hour?]

30-second loop: Explore → Encounter → Fight → Loot
5-minute loop:  Clear room → Choose path → Find shop → Upgrade
1-hour loop:    Complete run → Unlock permanent upgrades → Start new run

## 3. Mechanics
### 3.1 Movement
### 3.2 Combat
### 3.3 Progression
### 3.4 Economy

## 4. Content
### 4.1 Characters / Classes
### 4.2 Enemies
### 4.3 Items / Equipment
### 4.4 Levels / Worlds
### 4.5 Story / Narrative

## 5. Art Direction
- **Style:** [e.g., Pixel art, Low-poly, Realistic]
- **Color Palette:** [Primary colors and mood]
- **Reference Games:** [Visual references]
- **UI Style:** [Minimal, Skeuomorphic, Flat]

## 6. Audio Direction
- **Music Style:** [e.g., Chiptune, Orchestral, Ambient]
- **SFX Approach:** [Realistic, Stylized, Retro]

## 7. Technical Requirements
- **Engine:** Unity [version]
- **Min Spec:** [e.g., iPhone 8, Android 7+]
- **Target FPS:** [30/60]
- **Network:** [Offline / Online / Multiplayer]

## 8. Monetization
- **Model:** [Premium / F2P / Subscription]
- **IAP:** [What can players buy?]
- **Ads:** [If any, what type?]

## 9. Milestones
| Phase | Duration | Deliverable |
|-------|----------|-------------|
| Prototype | 2 weeks | Core loop playable |
| Alpha | 6 weeks | All mechanics, placeholder art |
| Beta | 4 weeks | Final art, balancing, testing |
| Launch | 2 weeks | Polish, marketing, submission |

## 10. Risks
| Risk | Impact | Mitigation |
|------|--------|------------|
| Scope creep | High | Strict feature freeze at Alpha |
| Performance on low-end | Medium | Profile early, budget per frame |
| Player retention | High | Playtest core loop in prototype |
```

## Mechanics Analysis Framework

When analyzing a game mechanic, evaluate:

| Dimension | Question |
|-----------|----------|
| **Depth** | How many meaningful decisions does the player make? |
| **Clarity** | Can the player understand the outcome of their action? |
| **Feedback** | Does the game respond immediately and satisfyingly? |
| **Mastery** | Can skilled players perform noticeably better? |
| **Variety** | Does the mechanic stay fresh over time? |
| **Integration** | Does it connect to other systems meaningfully? |
